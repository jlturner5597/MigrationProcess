Purpose & Use

Use this as a working playbook: concrete steps, explicit artifacts, and decision points. It assumes familiarity with Surpass modules and APIs.

Outputs across the project: signed plan, structure specs, data maps & schemas, migration runbook, validation report, cutover checklist, and handover pack.

Appendix

CLI skeleton (example) 
surpass-cli auth --client-id <id> --secret ****
surpass-cli load items --file items.jsonl --batch 500 --idem-key items-2025-08-08
surpass-cli load media --dir ./media --dedupe
surpass-cli jobs watch --since "2025-08-08T09:00Z" --until done
surpass-cli report reconcile --source snapshot.csv --target api --out recon.csv
Validation SQL snippets (illustrative) 
-- Count by subject
SELECT subject_id, COUNT(*) FROM items GROUP BY 1;

-- Detect orphaned refs
SELECT i.id
FROM items i LEFT JOIN media m ON m.id = i.media_id
WHERE m.id IS NULL;
